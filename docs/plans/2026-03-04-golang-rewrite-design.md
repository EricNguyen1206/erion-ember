# Erion Ember v3 — Go Rewrite Design

**Date**: 2026-03-04  
**Status**: Approved  
**Author**: brainstorming session

## Background

Erion Ember v2 is a TypeScript/Bun MCP Server for LLM Semantic Cache. This document describes a complete rewrite in Go as a **standalone gRPC + REST service** — deployable like Redis, not tied to the MCP protocol.

Core motivations:
- Eliminate Node.js/Bun runtime dependency
- Single static binary, zero external runtime
- gRPC as primary protocol (binary, low-latency — analogous to Redis RESP)
- Strict fast-path/slow-path separation: exact cache hits must never be blocked by ONNX inference

## Architecture

### Single Binary + ONNX Worker Pool

```
┌─────────────────────────────────────────────────────────┐
│                    Single Binary                        │
│                                                         │
│  gRPC (:50051) ◄──── grpc-gateway (:8080) REST/JSON    │
│        │                                                │
│        ▼                                                │
│  ┌─────────────┐                                        │
│  │ gRPC Handler │                                       │
│  └──────┬──────┘                                        │
│         │                                               │
│  ┌──────▼──────────────────────────────────┐            │
│  │         SemanticCache                   │            │
│  │  1. xxhash(normalize(prompt))           │            │
│  │     → LRU lookup → HIT → return ~0.1ms │            │
│  │  2. MISS → push to EmbedJob chan        │            │
│  │     → Worker Pool (N goroutines ONNX)  │            │
│  │     → HNSW cosine search               │            │
│  │     → return result                    │            │
│  └─────────────────────────────────────────┘            │
└─────────────────────────────────────────────────────────┘
```

**Design rationale**: In a Semantic Cache, `xxhash(text)` RAM lookup returns in ~0.1ms and bypasses ONNX entirely. If ONNX runs synchronously on the same goroutine as the gRPC handler, "miss" requests drag down the entire goroutine pool — hurting "hit" request latency. The Worker Pool isolates this slow path.

## API (Proto Definition)

```proto
syntax = "proto3";
package ember.v1;
option go_package = "github.com/EricNguyen1206/erion-ember/gen/ember/v1";

import "google/api/annotations.proto";

service CacheService {
  rpc Get    (GetRequest)    returns (GetResponse) {
    option (google.api.http) = { post: "/v1/cache/get", body: "*" };
  }
  rpc Set    (SetRequest)    returns (SetResponse) {
    option (google.api.http) = { post: "/v1/cache/set", body: "*" };
  }
  rpc Delete (DeleteRequest) returns (DeleteResponse) {
    option (google.api.http) = { post: "/v1/cache/delete", body: "*" };
  }
  rpc Stats  (StatsRequest)  returns (StatsResponse) {
    option (google.api.http) = { get:  "/v1/stats" };
  }
}

message GetRequest {
  string prompt = 1;
  optional float similarity_threshold = 2;
}
message GetResponse {
  bool   hit         = 1;
  string response    = 2;
  float  similarity  = 3;
  bool   exact_match = 4;
}

message SetRequest {
  string prompt   = 1;
  string response = 2;
  optional int32  ttl = 3;  // seconds
}
message SetResponse { string id = 1; }

message DeleteRequest { string prompt = 1; }
message DeleteResponse { bool deleted = 1; }

message StatsRequest {}
message StatsResponse {
  int64  total_entries     = 1;
  int64  cache_hits        = 2;
  int64  cache_misses      = 3;
  float  hit_rate          = 4;
  string compression_ratio = 5;
}
```

## Data Flow

### Get (read path)

```
gRPC Get(prompt)
  │
  ▼
normalizer.Normalize(prompt)            — lowercase, trim, collapse whitespace
  │
  ▼
xxhash.Sum64(normalized)                — uint64 key
  │
  ▼
metadataStore.FindByHash(key)
  │
  ├─ HIT  ──► lz4.Decompress(compressedResponse)
  │            └─ GetResponse{hit:true, similarity:1.0, exact_match:true}  (~0.1ms)
  │
  └─ MISS ──► EmbedJob{prompt, respChan: make(chan EmbedResult, 1)}
               │
               ▼  (non-blocking send to buffered channel)
            embedPool.jobChan <- job
               │
               ▼  (worker goroutine picks up)
            hugot.Embed(prompt)         — ONNX inference, all-MiniLM-L6-v2, 384-dim
               │
               ▼
            int8Quantize(vec)           — reduces memory 4x
               │
               ▼
            hnsw.SearchKNN(vec, k=5)
               │
               ├─ similarity >= threshold → GetResponse{hit:true, ...}
               └─ no match              → GetResponse{hit:false}
```

### Set (write path)

```
gRPC Set(prompt, response, ttl?)
  │
  ├─ embedPool.EmbedSync(prompt)        — blocking embed job (dedicated sync path)
  ├─ hnsw.AddItem(int8(vec))            — → vectorId
  ├─ lz4.Compress(prompt)
  ├─ lz4.Compress(response)
  └─ metadataStore.Set(xxhash, metadata, ttl)   — LRU insert
```

## Worker Pool Design

```go
type EmbedJob struct {
    prompt   string
    respChan chan<- EmbedResult
}

type EmbedPool struct {
    jobChan chan EmbedJob  // buffered: 512
    workers int            // default: runtime.NumCPU()
}

// Each worker holds its own ONNX session to avoid lock contention.
// hugot sessions are goroutine-safe when used independently.
func (p *EmbedPool) worker(model *hugot.Pipeline) {
    for job := range p.jobChan {
        vec, err := model.Embed(job.prompt)
        job.respChan <- EmbedResult{vec: vec, err: err}
    }
}
```

**Configuration defaults:**
| Parameter | Default | Notes |
|---|---|---|
| `EMBED_WORKERS` | `runtime.NumCPU()` | One ONNX session per worker |
| `EMBED_QUEUE_SIZE` | `512` | Buffered channel depth |
| `GRPC_PORT` | `50051` | Primary protocol |
| `HTTP_PORT` | `8080` | grpc-gateway REST |
| `CACHE_SIMILARITY_THRESHOLD` | `0.85` | Configurable per-request |
| `CACHE_MAX_ELEMENTS` | `100000` | HNSW max capacity |
| `CACHE_DEFAULT_TTL` | `3600` | Seconds |
